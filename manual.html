1. 概要

dirtracker は、Linux の設定ディレクトリを監視し、ファイル変更履歴を SQLite に保存するツールです。 Web UI で履歴の確認、差分表示、最新版または指定スナップショットを ZIP でエクスポートできます。

  • 監視対象例: /etc（必要に応じて /home などを追加可能）
  • 記録内容: 変更時刻、イベント種別、ファイル全文、前回との差分
  • 出力: フォルダ構造を保持した ZIP

2. 構成

レイヤー内容
Watcherinotify でファイルイベントを監視し、一定間隔でバックエンドへ送信
BackendFastAPI で履歴保存 API と Web UI を提供
DatabaseSQLite（./data/dirtracker.db）に履歴を蓄積
Frontend3 ペイン UI（ファイル一覧 / 変更履歴 / 差分・全文表示）
外部ネットワークアクセスは Docker の internal network で制限され、サービス間通信のみ許可されます。

3. 起動手順

3-1. 前提条件

  • Docker と Docker Compose が利用可能であること
  • 監視対象ディレクトリを読み取り専用でマウント可能であること

3-2. 起動

cd dirtracker
docker compose up --build

起動後、ブラウザで http://localhost:8000 にアクセスします。

3-3. 監視対象を追加する

docker-compose.yml の watcher 設定で、volumes と WATCH_DIRS を対応させて追加します。

watcher:
  volumes:
    - /etc:/watch/etc:ro
    - /home:/watch/home:ro
  environment:
    - WATCH_DIRS=/watch/etc,/watch/home
    - WATCH_PREFIX_STRIP=/watch
注意: WATCH_DIRS と volumes が一致しない場合、監視が開始されません。

4. 日常運用

4-1. ログ確認

docker compose logs -f backend
docker compose logs -f watcher

4-2. 停止・再起動

docker compose stop
docker compose start

4-3. 完全停止

docker compose down

4-4. バッチ送信間隔の調整

Watcher は既定で 60 秒ごとにイベントをまとめて送信します。短くする場合は環境変数を追加します。

environment:
  - BATCH_WINDOW_SECONDS=30

5. 画面の使い方

5-1. 左ペイン: ファイル一覧

  • クリックで対象ファイルを選択
  • チェックボックスで複数ファイルを選択し、まとめてエクスポート

5-2. 中央ペイン: 変更履歴

  • 時系列（新しい順）でスナップショットを表示
  • イベント種別: modified / created / deleted

5-3. 右ペイン: コンテンツ表示

  • 「差分」: 直前スナップショットとの差分を表示
  • 「全文」: 当該時点のファイル全文を表示

5-4. ヘッダー操作

  • 選択をエクスポート: チェック済みファイルの最新版を ZIP 化
  • 全ファイルをエクスポート: 全ファイルの最新版を ZIP 化
  • 更新: 一覧を即時再取得（自動更新は 15 秒ごと）

6. API リファレンス

MethodPath用途
GET/api/files監視ファイル一覧を取得
GET/api/files/{id}/snapshotsファイル単位の履歴取得
GET/api/snapshots/{id}スナップショット全文取得
GET/api/snapshots/{id}/diff差分取得
POST/api/ingest単発イベント取り込み
POST/api/ingest/batchバッチイベント取り込み
GET/api/exportZIP エクスポート

6-1. エクスポート API

  • /api/export: 全ファイル最新版
  • /api/export?file_ids=1,2: 指定ファイル最新版
  • /api/export?snapshot_ids=10,11: 指定時点の内容

7. バックアップと復旧

7-1. バックアップ

履歴データは ./data/dirtracker.db に保存されます。定期的にコピーを取得してください。

cp ./data/dirtracker.db ./data/dirtracker.db.bak

7-2. 復旧

サービス停止後に DB ファイルを戻して再起動します。

docker compose down
      cp ./data/dirtracker.db.bak ./data/dirtracker.db
docker compose up -d
cp ./data/dirtracker.db.bak ./data/dirtracker.db docker compose up -d

8. トラブルシュート

8-1. 画面にファイルが出ない

  • 監視対象で実際に変更が発生しているか確認
  • Watcher ログで監視開始メッセージとエラーを確認
  • WATCH_DIRS とマウント設定の不一致がないか確認

8-2. 差分が空になる

  • 初回スナップショットは比較元がないため差分なし
  • 同一内容で更新された場合も差分が空になる場合あり

8-3. バイナリファイルが記録されない

Watcher は UTF-8 テキストを対象にしており、バイナリや読み取り不可ファイルはスキップします。

8-4. LAN から UI に接続したい

既定では localhost のみ公開です。必要時は起動前に環境変数を指定します。

UI_BIND_HOST=0.0.0.0 docker compose up -d
